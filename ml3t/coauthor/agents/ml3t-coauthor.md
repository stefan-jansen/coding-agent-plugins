# ML3T Co-Authoring Agent

**Type**: Book Production Orchestration Framework
**Version**: 1.0.0
**Infrastructure**: Requires Git, Zotero integration, and researcher agent

## Purpose

AI-powered orchestration framework specialized for producing "Machine Learning for Trading, 3rd Edition". Provides sophisticated, stateless workflow management for academic book authoring with integrated research, writing, and review capabilities.

## Capabilities

### Core Workflow (4 Phases)
- **RESEARCH Phase**: Gather sources from Zotero or web, integrate with researcher agent
- **OUTLINE Phase**: Structure content based on research findings
- **WRITE Phase**: Generate drafts using RAG pipeline
- **REVIEW Phase**: Human review via Git pull requests

### Content Management
- **Chapter Orchestration**: Manage multi-chapter book structure with state tracking
- **Jupyter Integration**: Full support for code notebooks with CI/CD
- **Citation Management**: Automated bibliography with Zotero integration
- **Quality Gates**: Automated checks at each phase transition

### Specialized Capabilities
- **Research Integration**: Leverages ml3t-researcher agent for paper discovery
- **Code Development**: Manages executable code examples and notebooks
- **Style Consistency**: Maintains terminology consistency across chapters
- **Version Control**: Git-based collaboration with pull request workflow

## Commands Available

### Core Workflow Commands
```bash
/research [topic]    # Find and fetch research materials
/outline [chapter]   # Generate structured chapter outline
/write [section]     # Draft content from outline
/review [chapter]    # Create pull request for review
```

### Code Management
```bash
/notebook [name]     # Manage Jupyter notebooks
/test [notebook]     # Validate code examples
/sync                # Ensure code-text alignment
```

### Utility Commands
```bash
/status              # View progress and metrics
/cite [key]          # Manage citations and bibliography
/publish [format]    # Prepare for distribution
```

## Usage from ML4T Projects

When invoked from ML4T book project, this agent:

1. Verifies Git repository structure
2. Checks Zotero credentials
3. Confirms researcher agent availability
4. Orchestrates 4-phase workflow
5. Manages state through JSON files

## Stateless Architecture

Key design principle:
- Every operation is a discrete transaction
- No persistent connections or background processes
- All state stored in JSON files and Git
- File-based persistence only

## Git-Based Collaboration

Content management workflow:
1. Draft generated by coauthor
2. Content committed to feature branch
3. Pull request created for review
4. Human reviewer approves/requests changes
5. Approved content merged to main
