# Clear Writing - Complete Reference

## Excellent Writing: The 15 Principles

### Principle 1: No Filler Words
Remove words that add no information.
- "It's really important to note that..." -> Just state it
- "In order to achieve this goal" -> "To achieve this"
- "At this point in time" -> "Now"

### Principle 2: Information-Dense Sentences
Every sentence adds new information. No repetition.
- "The framework is useful. It helps with development." -> "The framework accelerates development."

### Principle 3: Context Without Preamble
Provide background efficiently, then get to the point.
- "Before we dive in, let's understand..." -> Just explain it

### Principle 4: Economy of Language
Shorter words beat longer words. Fewer words beat more.
- utilize -> use
- facilitate -> enable
- in close proximity to -> near

### Principle 5: One Example Suffices
If one clear example makes the point, don't add more.

### Principle 6: Neutral Questions
Questions should be genuinely neutral, not telegraph the answer.
- "Isn't it obvious that X is better?" -> "Which approach is better?"

### Principle 7: No Restating Premise
After introducing a concept, don't explain why it matters. Trust reader.

### Principle 8: Cut Transitional Fluff
Avoid sentences that announce importance. Jump to content.
- "This is important because..." -> Just state what makes it important

### Principle 9: Combine Related Ideas
Merge related statements instead of repeating concepts.

### Principle 10: Trust Reader Intelligence
Don't over-explain obvious implications. Assume competence.

### Principle 11: Start Sections With Meat
Lead with specific advice, not general importance statements.

### Principle 12: Simple Punctuation
Use periods, commas, colons instead of em dashes unless truly needed.

### Principle 13: Cut Qualifying Phrases
Remove phrases that don't add concrete information.
- "when done correctly" -> Either explain how or trust reader

### Principle 14: Direct Statements
State what things do, not that they're important.
- "X is important because..." -> "X does [specific thing]"

### Principle 15: Remove Setup Phrases
Delete phrases like "It's worth noting that" and state the point directly.

## Plain Language: The Four Rules

### Rule 1: Use Active Voice (>80% target)

**Passive** (subject receives action): "The error was caught by the test"
**Active** (subject performs action): "The test caught the error"

**Detection pattern**: "was/were/is/are + past participle + by"

**When passive is okay**:
- Unknown actor: "The system was compromised"
- Actor irrelevant: "Errors are logged automatically"

### Rule 2: Shorten Sentences (<25 words average)

**Targets**:
- Average: <25 words per sentence
- Maximum: Avoid >35 word sentences
- Variety: Mix short (10-15) and medium (20-25)

**How to shorten**:
1. Split at "and", "but", "because"
2. Remove redundancy
3. Break into bullets if listing >3 items

### Rule 3: Eliminate Jargon (or Explain on First Use)

**When jargon is okay**:
- Technical audience expects it
- No simpler alternative exists
- Explained on first use

**When to avoid**:
- Simpler word exists: "utilize" -> "use"
- Buzzword adds no meaning: "leverage synergies"

### Rule 4: Write Clearly

**Nominalization** (verb -> noun weakens writing):
- make a decision -> decide
- perform an analysis -> analyze
- give consideration to -> consider

**Weak verbs**:
- utilize -> use
- facilitate -> enable
- implement -> build

**Redundancy**:
- advance planning -> planning
- past history -> history
- end result -> result

## Information Mapping: The Four Pillars

### Pillar 1: Chunking (5-9 Rule)

Group information into 5-9 item chunks matching working memory.

**Application**:
- Lists: 5-9 items max; >9 items -> create categories
- Paragraphs: 3-6 sentences typically
- Sections: 3-5 paragraphs before subheading

### Pillar 2: Labeling (Descriptive Headings)

Headings describe content, not generic placeholders.
- "Introduction", "Overview", "Section 2" (bad)
- "Why Progressive Disclosure Matters", "Three Common Mistakes" (good)

### Pillar 3: Visual Hierarchy (Consistent Structure)

Heading levels create clear document structure; no skipped levels.
- H1: Document title (one per document)
- H2: Major sections
- H3: Subsections within H2
- Never skip: H1 -> H3 (skip H2)

### Pillar 4: Scannability (F-Pattern Optimization)

Readers read first 2 paragraphs, then scan left edge.

**Optimization**:
1. Front-load important info
2. Descriptive first words
3. Keywords in headings
4. Bold key terms
5. Lists over prose

## Application by Content Type

| Type | Active Voice | Sentence Length | Jargon | Paragraphs |
|------|-------------|-----------------|--------|------------|
| Tutorial | >90% | <20 words | None | 2-3 sentences |
| How-To | >85% | <22 words | Defined | 3-4 sentences |
| Reference | >70% | <25 words | Expected | 4-6 sentences |
| Explanation | >80% | <25 words | Explained | 4-6 sentences |

## Quality Metrics

**Automated detection**:
- Filler words: "really", "very", "actually", "basically"
- Em dash frequency
- Setup phrases: "It's worth noting", "The key point is"
- Passive voice markers

**Manual review**:
- Information density (check for repetition)
- Section starts with meat
- Direct statements (substance, not importance claims)
