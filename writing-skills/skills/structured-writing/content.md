# Structured Writing - Complete Reference

## Pyramid Principle (Full)

### The Pyramid Structure

```
                  [ANSWER / Core Message]
                           |
        +------------------+------------------+
        |                  |                  |
   [Argument 1]      [Argument 2]      [Argument 3]
        |                  |                  |
    +---+---+          +---+---+          +---+---+
    |   |   |          |   |   |          |   |   |
  [Details] [Details] [Details] [Details] [Details] [Details]
```

**Levels**:
1. Level 1: The answer/conclusion (what reader needs to know)
2. Level 2: Major supporting arguments (why answer is true)
3. Level 3: Evidence and examples (what proves arguments)
4. Level 4: Details and elaboration (depth as needed)

### Three Core Rules

**Rule 1: Answer First**
- Start with conclusion
- Supporting details provide confidence
- Reader can stop at any level and understand core idea

**Rule 2: Group Related Ideas**
- Ideas in each group must be:
  - Related to each other (same category)
  - Support the idea above them
  - At same level of abstraction

**Rule 3: Logical Order**
- Structural: Parts of something
- Chronological: Time sequence
- Comparative: Ranking/comparison
- Problem-solution: Issue then resolution

### Application Examples

**Business Communication**:
```markdown
**Recommendation**: [The answer - one sentence]

**Rationale**: [3-5 supporting arguments]
1. Argument 1
2. Argument 2
3. Argument 3

**Details**: [Evidence for each argument]
```

**Technical Explanation**:
```markdown
**[Core message - what it is]**

How it works:
1. Component/step 1
2. Component/step 2
3. Component/step 3

Why it matters:
- Benefit 1
- Benefit 2
```

## SCQA Framework (Full)

### Four Elements

**1. Situation (S)**
- The stable, uncontroversial starting point
- Establishes common ground
- Non-controversial (reader nods along)

**2. Complication (C)**
- The problem, change, or challenge
- Creates tension, makes reader care
- Resonates with audience pain

**3. Question (Q)**
- What reader now wants answered
- Natural from complication
- Often implicit (can omit if obvious)

**4. Answer (A)**
- Your solution, recommendation, core message
- Directly addresses question
- Concise (details follow in body)

### Complete Example

**Topic**: Introducing CAF

**Situation**:
> Claude Code has become a popular AI coding assistant, with developers using it daily.

**Complication**:
> However, Claude Code out-of-box is a generic assistant. It doesn't understand your domain, workflows, or constraints. As projects grow complex, generic responses frustrate.

**Question** (implicit):
> How can we customize Claude Code for our specific domain?

**Answer**:
> The Claude Agent Framework (CAF) transforms Claude Code into domain-specific agents through markdown-based customization.

### Variations

**SCQ (Implicit Answer)**: Answer is your entire document
**SCA (Implicit Question)**: Question obvious from complication
**CSA (Situation Assumed)**: Jump to complication when audience knows context

### Combining with Pyramid

```markdown
Opening (SCQA):
+- Situation
+- Complication
+- Question
+- Answer [This is Level 1 of pyramid]

Body (Pyramid):
+- Argument 1 supporting Answer [Level 2]
|  +- Evidence [Level 3]
+- Argument 2 supporting Answer [Level 2]
|  +- Evidence [Level 3]
+- Argument 3 supporting Answer [Level 2]
   +- Evidence [Level 3]

Closing:
+- Reinforce Answer + CTA
```

## Topic Sentence Method (Full)

### Five Patterns

**Pattern 1: Definition + Importance**
- Formula: `[Concept] is [definition]. This [importance/impact]...`
- Use for: Introducing concepts, frameworks, principles

**Pattern 2: Action + Result**
- Formula: `[Action verb] [object] to [achieve result]. This [outcome]...`
- Use for: Instructions, procedures, recommendations

**Pattern 3: Problem + Solution**
- Formula: `[Problem] occurs when [context]. [Solution] resolves this by [mechanism]...`
- Use for: Troubleshooting, debugging, design rationale

**Pattern 4: Concept + Analogy**
- Formula: `[Abstract concept] [definition]. Like [familiar analogy], [mapping]...`
- Use for: Explaining abstractions, teaching beginners

**Pattern 5: Question + Answer**
- Formula: `[Question]? [Answer]. [Elaboration/Evidence]...`
- Use for: Anticipating reader questions, addressing objections

### Pattern Selection by Content Type

**Tutorial**: Action + Result, Concept + Analogy
**How-To**: Action + Result, Problem + Solution
**Reference**: Definition + Importance, Problem + Solution
**Explanation**: All patterns, especially Concept + Analogy

### Quality Criteria for Topic Sentences

- **Explicit**: States the point clearly, no implicit assumptions
- **Connected**: Links to parent message in hierarchy
- **Standalone**: Makes sense when read in isolation (scannability)
- **Evidence-Ready**: Makes falsifiable claims evidence can support
- **Focused**: Single main point per sentence

### Common Pitfalls

1. **Topic Sentence as Transition**: "Now let's discuss..." -> State the point instead
2. **Burying the Lead**: Put conclusion first, details second
3. **False Specificity**: "High accuracy" -> State actual numbers
4. **Multiple Claims**: One claim per topic sentence
5. **Disconnected from Hierarchy**: Every topic sentence advances parent claim
